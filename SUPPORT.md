# Support

Use the right channel for the right kind of problem.

## Bugs

Open an issue with the bug template and include:

- The model backend.
- Your feature flags.
- Your platform.
- Whether you used local files or Hugging Face download fallback.
- A small reproduction snippet when possible.

## Feature requests

Open an issue with the feature request template. Be explicit about whether the request is:

- A new model backend.
- A missing upstream capability in an existing backend.
- A usability improvement in config, docs, or examples.

## Usage questions

Open a normal issue if you are blocked and the answer is likely to help the next user too.

## Security reports

Do not file public issues for vulnerabilities. Use [SECURITY.md](SECURITY.md).

## License questions

Remember that crate licensing and model licensing are separate. If your question is about model deployment rights, check the upstream model card first and include the exact checkpoint you are using.
